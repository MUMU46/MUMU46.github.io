Chain of Thought

Building a systems with the ChatGPT AI

课程链接🔗

🧠 Chain-of-Thought（思维链）是什么？

这是一个提示工程（prompt engineering）技巧，核心思想是：

  • 让模型在回答用户问题前，先分步骤分析和推理。
  • 使用 结构化模板 指导模型一步步思考，再输出最终回答。
  • 这样做可以显著提高模型在复杂任务中的表现，避免跳步或误解用户意图。

实践内容：

我们使用了一个关于顾客咨询产品的问题案例，分步骤让模型分析并回复。具体流程如下：

1. 环境准备

导入 openai 库，加载 API 密钥（通过 .env 文件），并封装了一个函数：

这个函数用来向 OpenAI 的聊天模型发送信息并获取回复。

2. 系统提示（System Prompt）设计：

我们设计了一个详细的系统提示 system_message，用 5个步骤指导模型处理客户咨询：

📋 规则流程（每一步用 #### 作为分隔符）：
  1. 判断用户是否在问特定产品（不是问产品类别，而是明确的产品名）。
  2. 检查提到的产品是否在店铺提供的5个产品列表中。
  3. 识别用户是否做了某些假设（如某个笔记本更大、某个有更长保修期）。
  4. 验证这些假设是否正确（基于产品信息判断）。
  5. 礼貌地纠正错误并回复用户，只提店里有卖的产品。

  • 判断是不是问产品 → 是

  • 检查是否正确 → 错，Chromebook更便宜
  • 回复用户并纠正：TechPro Desktop其实更贵

4. 隐藏推理过程（Inner Monologue）：

我们可以通过代码 response.split(delimiter)[-1].strip() 只保留最后一步的用户可见回应，隐藏模型中间的“思维过程”。

这样可以：

  • 模型内部逐步推理
  • 但用户只看到简洁友好的回答
