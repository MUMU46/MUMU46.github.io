unordered_set

实现哈希表

查找一个数在数组中是否存在

使用哈希表查找可以实现O(1)的时间复杂度

unorderd_map

  • unordered_map是一个将key和value关联起来的容器，它可以高效的根据单个key值查找对应的value。

  • key值应该是唯一的，key和value的数据类型可以不相同。

  • unordered_map存储元素时是没有顺序的，只是根据key的哈希值，将元素存在指定位置，所以根据key查找单个value时非常高效，平均可以在常数时间内完成。

    map

    map和unordered_map的区别:

    1. 底层数据结构：

      • unordered_map 使用哈希表作为底层数据结构。哈希表通过哈希函数将键映射到存储桶，以实现快速的查找、插入和删除。
      • map 使用红黑树作为底层数据结构。红黑树是一种自平衡的二叉搜索树，能够保持元素的有序性，支持按照键进行排序。

    2. 有序性：

      • unordered_map 不保证元素的有序性，元素的存储顺序可能会随着插入和删除操作而改变。
      • map 保持元素的有序性，元素按照键的自然顺序进行排序，或者可以通过自定义比较器进行排序。

    3. 性能特征：

      • unordered_map 提供了平均 O(1) 的时间复杂度用于插入、删除和查找操作，但在最坏情况下可能会达到 O(n)，其中 n 是容器中的元素数量。
      • map 提供了平均 O(log n) 的时间复杂度用于插入、删除和查找操作，红黑树的自平衡特性保证了较稳定的性能表现。

    4. 空间复杂度：

      • unordered_map 和 map 的空间复杂度都是 O(n)，其中 n 是容器中的元素数量。但是由于哈希表的冲突问题，unordered_map 可能需要更多的额外空间来处理哈希冲突。

  • 如果需要快速的插入、删除和查找操作，并且不关心元素的顺序，可以选择 unordered_map。

  • 如果需要保持元素的有序性，或者需要在有序数据上进行操作，可以选择 map。
