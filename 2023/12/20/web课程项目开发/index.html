Web课程实验报告

实验目标

使用html、css、javascript、php开发一个web系统

实验内容

taskforum–任务管理及打卡平台

技术栈

使用react + Laravel + Inertia

  • react: 是一个自由及开放源代码的前端JavaScript工具库， 用于基于UI组件构建用户界面。
  • Laravel: 是Taylor Otwell 开发的一款基于PHP 语言的Web 开源框架，采用了MVC 的架构模式.
  • Interia: Inertia.js 是一种现代的服务端渲染 (SSR) 技术，它允许在 Laravel 应用中使用前端框架（如 React）而无需分离两个项目。Inertia 可以桥接Laravel 应用程序和现代 Vue 或 React 前端，可以使用 Vue 或 React 构建完整的现代前端, 同时利用 Laravel 路由和控制器进行路由、数据注入和身份验证 - 所有这些都在单个代码存储库中完成。
  • 本系统采用前后端分离, 但通过interia实现在一个项目中管理前后端代码, 为开发带来更大的便利.

开发平台

PhpStorm

系统需求

该系统的主要功能是用户可以创建账户添加和保存自己的待办清单以及打卡任务.

  • 待办清单:
    • 用户可以新增、修改、完成、删除、筛选清单的内容;
  • 任务打卡:
    • 用户可以新增、删除需要定期打卡的任务;
    • 新增任务需要提供的内容: 打卡标题、内容、打卡周期、是否公开;
    • 每个周期内未打卡的任务会显示剩余打卡时间,以提醒用户;
    • 每个任务中还会显示已打卡次数;
  • 论坛:
    • 打卡任务共享: 用户公开发布的打卡任务可以在首页论坛被其他用户看到, 并且他们可以选择加入自己的打卡列表里
  • 个人信息管理:
    • 用户可以在个人主页管理修改自己的账号信息
    • 用户登录及登出

实验总结

过去我一直开发的都是前后端分离的项目, 有分别使用react和springboot开发前端和后端的经验.

因此本次实验继续沿用前后端分离的模式, 而这是我第一次使用php开发后端, 学习并使用了lavarel框架,同时为了更好集成前后端代码, 通过Interia实现在lavarel后端项目中使用前端框架, 从而不必维护两个独立的项目, 使前后端的开发更加集成和简化.

其次, 实验中我也遇到一些问题, 比如对于自己提出的打卡功能之前没有开发过, 我从数据库建库, 建表, 到后端业务逻辑的编写, 最后到前端页面的实现, 在这个过程中一步步完成新功能的开发. 对于之前已经有过开发经验的用户管理及论坛功能则比较得心应手. 整体来说, 这次实0验收获了不少, 丰富了我的全栈开发项目经验, 从语法学习到系统开发, 掌握并实践了新的语言php.

全新的技术Lavarel和interia学习也丰富了我的学习经验, 让自己能更熟练地学习和接受新的技术和框架.
